# GaokaoAgent 前端优化报告

**优化时间**: 2026-01-01
**优化目标**: 提升用户交互体验，实现真实数据驱动的可视化

---

## 一、问题诊断

### 修复前的问题
1. **假进度模拟** - 随机显示步骤消息，与实际执行不符
2. **纯 Markdown 展示** - 无法结构化展示博弈矩阵（志愿表）
3. **无交互功能** - 无法筛选、排序、查看详情
4. **缺少可视化** - 录取概率、策略标签等关键信息展示不直观

---

## 二、优化方案

### 1. 后端 API 增强

**文件**: `backend/src/main.py`

**新增返回字段**:
```python
class QueryResponse(BaseModel):
    success: bool
    message: str
    report: Optional[str] = None
    research_report: Optional[str] = None
    intent_type: Optional[str] = None  # 意图类型 (quant/research/multimodal)
    loop_type: Optional[str] = None  # 循环类型 (fast/slow/multimodal)
    game_matrix: Optional[Dict[str, Any]] = None  # ✨ 新增：结构化博弈矩阵
    user_profile: Optional[Dict[str, Any]] = None  # ✨ 新增：用户画像
    debug_logs: list = []
```

**核心改进**:
- 返回完整的 `game_matrix` JSON 数据（包含所有志愿、概率、策略标签）
- 返回 `user_profile` 数据（分数、位次、选科）
- 返回 `intent_type` 和 `loop_type`，便于前端展示

### 2. 前端组件创建

#### 组件 1: `ProgressTracker.tsx` (真实进度追踪)

**功能**:
- 解析后端 `debug_logs`，显示真实的 Agent 执行步骤
- 展示意图类型（量化推荐 / 政策研究 / 图像解析）
- 展示循环类型（快思考 / 慢思考 / 多模态）
- 实时状态指示（pending / running / completed / failed）

**可视化元素**:
```tsx
✓ 绿色对勾 - 已完成
⚙ 旋转图标 - 进行中
✗ 红色叉号 - 失败
⏰ 灰色时钟 - 等待中
```

**Agent 映射**:
| Agent ID | 显示名称 |
|----------|---------|
| `meta_router` | 意图识别 |
| `profiling_agent` | 用户画像分析 |
| `game_agent` | 量化匹配引擎 |
| `report_agent` | 报告生成 |
| `critic_agent` | 质量审计 |
| `deep_research` | 深度研究 |
| `multimodal_parser` | 多模态解析 |

---

#### 组件 2: `GameMatrixView.tsx` (志愿表结构化展示)

**功能**:
- 结构化展示所有推荐志愿（冲/稳/保）
- 筛选功能：按策略标签筛选
- 排序功能：按录取概率或预测位次排序
- 卡片式展示：每个志愿一张卡片，包含所有关键信息

**卡片内容**:
1. **基础信息**
   - 学校名称
   - 专业名称
   - 策略标签（冲刺 🟠 / 稳妥 🔵 / 保底 🟢）

2. **核心指标**
   - 录取概率（百分比 + 进度条）
   - 预测最低位次
   - 位次置信区间（布林带）

3. **风险提示**
   - 波动性等级（低波动 / 中波动 / 高波动）
   - 调剂风险（⚠ 调剂概率 > 10% 时显示）
   - 黑名单警告（🚫 可能调剂到不喜欢的专业）

4. **舆情信息**
   - 新闻摘要（如有）

**交互功能**:
- **筛选**: 全部 / 冲刺 / 稳妥 / 保底
- **排序**: 录取概率 / 预测位次（升序/降序）
- **统计概览**:
  - 总数、冲刺数、稳妥数、保底数
  - 策略平衡指示（✓ 均衡 / ⚠ 待优化）

**颜色方案**:
| 策略标签 | 背景色 | 边框色 | 文字色 |
|---------|--------|--------|--------|
| 冲刺 (rush) | 橙色浅背景 | 橙色边框 | 橙色文字 |
| 稳妥 (target) | 蓝色浅背景 | 蓝色边框 | 蓝色文字 |
| 保底 (safe) | 绿色浅背景 | 绿色边框 | 绿色文字 |

### 3. 主应用更新

**文件**: `frontend/src/App.tsx`

**核心改进**:

1. **移除假进度模拟**
   ```tsx
   // 修复前（REMOVED）:
   const progressInterval = setInterval(() => {
     const steps = ["正在构建用户画像...", ...];
     setCurrentStep(steps[Math.floor(Math.random() * steps.length)]);
   }, 3000);

   // 修复后（NEW）:
   const steps = parseDebugLogsToSteps(analysisResult.debug_logs);
   setProgressSteps(steps);
   ```

2. **解析 Debug Logs 到 Agent Steps**
   ```tsx
   const parseDebugLogsToSteps = (logs: string[]): AgentStep[] => {
     const agentPattern = /\[(OK|WARN|ERROR)\]\s*([^:]+):\s*(.+)/;
     // 解析日志 -> 提取 Agent 名称 + 状态 + 消息
   }
   ```

3. **三段式渲染**
   - **分析前**: 显示表单
   - **分析中**: 显示加载动画 + 实时进度追踪
   - **分析后**: 显示进度总结 + 志愿表 + 报告

## 五、技术亮点

1. **数据驱动**
   - 前端不再伪造数据
   - 所有展示来自后端真实执行结果

2. **TypeScript 类型安全**
   - 完整的接口定义（AgentStep, GameMatrix, GameRow）
   - 编译时类型检查

3. **响应式设计**
   - Grid 布局自适应（1列 → 2列 → 3列）
   - 移动端友好

4. **可访问性**
   - 语义化 HTML
   - 明确的状态指示（图标 + 颜色）

5. **性能优化**
   - useMemo 缓存筛选/排序结果
   - 避免不必要的重渲染
