# 高考志愿填报、匹配机制与认知风险：研究综述

## 1. 核心问题

高考志愿填报不只是“根据往年分数线推荐学校”，而是一个由匹配机制、考生策略、信息不对称、群体预期和后悔风险共同构成的高风险决策市场。合理的项目定位应该从“录取概率预测”升级为“匹配机制约束下的认知与风险决策辅助”。

一句话记忆：

> 高考志愿不是静态推荐，而是在机制、信息和情绪共同作用下做风险选择。

## 2. 文献主线一：匹配机制、稳定性与合理妒忌

匹配理论的基本概念来自 Gale-Shapley 延迟接受机制。后续学校选择研究把录取机制分成即时接受机制、延迟接受机制和平行机制，并比较它们在策略性、稳定性和公平性上的差异。

在学校选择和高考录取中，“合理妒忌”是衡量匹配不稳定和不公平的重要指标。直观地说，如果学生 A 的优先级高于学生 B，却更喜欢 B 被录取的学校，那么 A 对 B 存在合理妒忌。没有合理妒忌通常意味着匹配更稳定、更公平。

| 文献 | 核心结论 | 对项目的启发 |
| --- | --- | --- |
| Gale & Shapley (1962) | 延迟接受机制给出稳定匹配的经典框架 | 高考录取可以被看作匹配问题，而不是普通排序问题 |
| Abdulkadiroglu & Sonmez (2003) | 学校选择机制需要兼顾策略激励和公平 | 推荐系统要解释机制约束，不只是给列表 |
| Pathak & Sonmez (2008) | Boston / IA 机制会让复杂策略伤害不熟练家庭 | “割韭菜感”可理解为机制复杂度下的信息弱势 |
| Chen & Kesten (2017) | 中国平行机制介于 IA 和 DA 之间，较 IA 更稳定、更少可操纵 | 平行志愿降低风险，但不等于没有策略问题 |
| Kang, Ha, Song & Zhou (2025 working paper) | 平行机制减少合理妒忌、落榜和复读，中等成绩学生改善更明显 | 中分段用户应作为重点风险人群 |

一句话记忆：

> 合理妒忌把“我是不是被不公平地挤掉了”变成可计算的匹配稳定性问题。

## 3. 重点论文：康乐等关于合理妒忌的研究

康乐、哈巍、宋阳、周森的论文《Matching Mechanisms, Justified Envy, and College Admission Outcomes》直接对应南京大学讲座主题。论文使用 2005-2011 年数百万考生个体数据，比较即时接受机制和平行机制下的合理妒忌与学生结果。

主要发现：

| 发现 | 含义 |
| --- | --- |
| 平行机制显著降低合理妒忌 | 机制改革提高了匹配稳定性 |
| 平行机制减少未录取和复读 | 机制设计影响真实教育结果，不只是理论公平 |
| 中等成绩学生合理妒忌更高，收益更大 | 中分段更容易被机制、策略和拥挤影响 |
| 平行志愿数量增加的效果不一定线性 | “多填几个”不必然等于更公平或更优 |
| 低 SES / 贫困地区学生合理妒忌更高，但改革缩小差距 | 信息和策略能力差异会转化为匹配不公平 |

项目启发：

> 系统不应只输出“你能上哪些学校”，还应输出“这个方案是否容易产生后悔、妒忌或被低分竞争者挤占的感受”，也就是 ex-post regret / justified envy proxy。

一句话记忆：

> 康乐这条线证明：志愿填报的核心风险之一是匹配稳定性，而不是单个学校概率。

## 4. 文献主线二：中国高考平行志愿改革的经验研究

中国高考录取机制提供了天然实验：不同省份在不同年份从顺序志愿 / 即时接受机制转向平行志愿机制。

| 文献 | 数据和方法 | 核心结论 |
| --- | --- | --- |
| Ha, Kang & Song (2020) | 全国 Tier I 高校录取面板，DID / event study | 从 IA 到平行机制提高匹配稳定性，第一平行志愿带宽更大时效果更强，但非线性 |
| Chen, Jiang & Kesten (2020) | 省份局部改革自然实验 | 平行机制后学生填更多学校，第一志愿更敢填高层次学校，匹配更稳定 |
| Bo, Liu, Shiu, Song & Zhou (2019) | 2005-2011 行政数据 | 平行机制约降低 6% 错配；知分填报约降低 18% 错配 |
| Pu (2021) | 广西、河北、四川反事实模型 | BM 与 DA 在福利上可能存在权衡，不是所有维度 DA 都更好 |
| Chen, Kesten 等动态机制研究 | 内蒙古动态机制，理论与实验 | 信息实时反馈能降低部分不确定性，但也引入策略和时间压力 |

项目启发：

> 你的系统可以把“机制理解”变成一个模块：先解释用户所在省份的志愿机制，再判断用户是否在用错误的机制直觉做决策。

一句话记忆：

> 平行志愿改善稳定性，但仍留下信息、策略和偏好表达问题。

## 5. 文献主线三：偏好、信息误差与策略错误

高考志愿的用户问题不只是“不知道能不能上”，还包括“不知道自己真正偏好什么”和“误解专业、学校、就业与风险”。这与偏好估计、信息干预和行为错误文献相连。

| 文献 | 核心结论 | 对项目的启发 |
| --- | --- | --- |
| Fack, Grenet & He (2019) | 即便在策略简单机制下，不能简单把申请列表当真实偏好 | 系统要区分 stated preference 和 inferred preference |
| Shorrer & Sovago (2017) | 在策略简单环境中，申请者仍会犯明显错误 | 不能假设用户理解机制和机会成本 |
| Wiswall & Zafar (2015) | 信息实验会改变学生对专业收益和能力的信念 | 专业误解纠偏可以改变选择 |
| Conlon (2017) | 学生对不同专业收入风险估计较差，风险信息会改变偏好 | 报告不能只给平均收入，要给风险分布 |
| Leighton & Speer (2024) | 专业选择受就业、课程难度、工作灵活性等多维信息影响 | 不能把“就业薪资”当唯一专业价值 |

项目启发：

> 应增加 `Preference Elicitation` 和 `Misconception Correction`：系统先识别用户对专业/学校的隐含假设，再判断这些假设是否来自误解、焦虑或热门叙事。

一句话记忆：

> 用户填的志愿不一定是真偏好，可能是信息误差和焦虑后的表面选择。

## 6. 文献主线四：羊群行为、信息级联与“热门专业加速期”

你提出的索罗斯“加速期”可以更学术地接到信息级联和反身性上。高考志愿市场中，往年分数线、机构推荐、短视频叙事和家长焦虑会相互强化：大家认为某专业热门，于是报考人数增加；报考增加推高录取位次；位次上升又被解释为“更有价值”，形成自我强化。

对应理论：

| 理论 | 对高考志愿的解释 |
| --- | --- |
| 信息级联 | 当个体不确定时，会观察前人的选择，往年录取线成为“他人行动”的信号 |
| 羊群行为 | 家长和考生追逐热门专业，可能忽视自身适配 |
| 反身性 | 群体对专业价值的认知会影响报考行为，报考行为又改变录取线这一“价格” |
| 情绪放大 | 滑档恐惧、浪费分焦虑、学校 title 焦虑会改变风险偏好 |

项目启发：

> 应加入 `Crowding / Reflexivity Risk`：识别某院校专业是否处于热度加速期，提醒用户区分“真实价值提升”和“报考拥挤推高”。

一句话记忆：

> 分数线上涨不一定代表价值上涨，也可能代表群体预期拥挤。

## 7. 与 GaokaoAgent 项目的结合

基于上述文献，项目可以从“推荐系统”改造成“四层决策 harness”：

| 层 | 目标 | 输出 |
| --- | --- | --- |
| 机制层 | 解释省份平行志愿规则、投档和退档逻辑 | `MechanismProfile` |
| 概率层 | 估计专业组录取概率、波动、冲稳保 | `GameMatrix` |
| 认知层 | 识别专业误解、学校 title 偏差、群体热度 | `PerceptionRisk` |
| 情绪层 | 识别滑档恐惧、浪费分焦虑、后悔风险 | `EmotionRisk` |
| 审计层 | 检查保底、黑名单、证据、合理妒忌 proxy | `AuditResult` |

一句话记忆：

> 下一版项目应从“录取概率推荐”升级为“机制 + 概率 + 认知 + 情绪 + 审计”的决策 harness。

## 8. 招生计划人数与专业组：不要把“稳”和“捡漏”混在一起

招生计划人数和专业组结构应该成为下一版推荐核心。直觉上，招生名额多的专业组往往更稳定，因为个体随机波动被更大的样本吸收；招生名额少的专业组更容易出现波动，因此既可能带来捡漏机会，也可能带来更高滑档风险。系统不能简单把“小计划”判成坏，也不能简单把“大计划”判成好，而要把它拆成两个信号：稳定性和机会性。

一句话记忆：

> 名额多代表稳定性，名额少代表波动性；波动既可能是风险，也可能是捡漏。

### 8.1 为什么专业组是最小分析单元？

新高考里，院校不是最小风险单位，专业组才是。一个学校的不同专业组可能有不同选科要求、不同专业组合、不同招生计划、不同调剂范围和不同录取位次。只按学校推荐会掩盖真正的风险。

| 分析粒度 | 问题 | 为什么不够 |
| --- | --- | --- |
| 学校 | 只知道学校层次和大致分数线 | 无法判断专业组内调剂风险 |
| 专业 | 看起来精确 | 实际投档常按专业组，不是单专业 |
| 专业组 | 同时包含选科、计划、调剂范围和历史位次 | 应作为推荐和风险评估的核心单位 |

项目口径：

> 我们不是推荐“某个学校”，而是推荐“某个院校专业组”，因为用户真正承担的是专业组录取和调剂风险。

一句话记忆：

> 学校是品牌，专业组才是投档风险单位。

### 8.2 小计划专业组为什么可能捡漏？

小计划专业组的核心特征是波动更大。招生少时，少数考生的选择变化就可能显著影响最低位次；如果该专业组名称冷门、信息不充分、往年波动大或被热门叙事遮蔽，就可能出现低估机会。

可能捡漏的条件：

| 条件 | 含义 |
| --- | --- |
| 招生计划少 | 分数线对少数考生行为更敏感 |
| 历史位次波动大 | 说明不是稳定热门资产 |
| 专业名称被误解 | 用户可能因为刻板印象回避 |
| 学校强但专业组不热门 | 学校品牌和专业组热度不一致 |
| 近年没有连续上移 | 不是已经被市场追热 |
| 与用户真实偏好匹配 | 捡漏不是为了便宜，而是适配但被低估 |

风险提示：

> 小计划不是天然捡漏。它只是更容易出现价格偏差，方向可能向上也可能向下。系统应该把它标成 high variance opportunity，而不是直接推荐用户冲。

一句话记忆：

> 小计划的价值在于波动，不在于一定更好。

### 8.3 大计划专业组为什么更稳定？

大计划专业组通常更接近“流动性好”的资产。名额多、样本大、历史线更平滑，单个考生行为对录取线影响较小，因此更适合做稳妥和保底。但大计划也可能因为热门专业组拥挤而被推高，不能只看人数。

稳定信号：

| 信号 | 解释 |
| --- | --- |
| quota 大 | 样本更大，随机波动被摊薄 |
| 历史最低位次标准差小 | 往年录取更稳定 |
| 招生计划连续 | 没有突然扩招或缩招 |
| 专业组内部专业差异小 | 调剂后体验落差较小 |
| 不处于热度加速期 | 没有明显群体追逐 |

一句话记忆：

> 大计划适合做安全垫，但仍要检查是否被热门叙事推高。

### 8.4 应该新增的指标

下一版 `GameMatrix` 里，每个 `MajorGroupRow` 不应只存 `admission_prob` 和 `strategy_tag`，还应该显式存招生规模和专业组结构风险。

建议字段：

```json
{
  "quota": 42,
  "quota_bucket": "small",
  "quota_stability_score": 0.38,
  "variance_opportunity_score": 0.72,
  "major_group_dispersion": 0.64,
  "adjustment_range_risk": 0.57,
  "crowding_adjusted_prob": 0.61,
  "recommendation_role": "rush_opportunity"
}
```

字段解释：

| 字段 | 含义 |
| --- | --- |
| `quota` | 该院校专业组招生计划人数 |
| `quota_bucket` | small / medium / large |
| `quota_stability_score` | 名额越多、历史波动越小，稳定性越高 |
| `variance_opportunity_score` | 名额少、波动大、未被追热时，机会分更高 |
| `major_group_dispersion` | 专业组内部专业差异，差异越大调剂不确定性越高 |
| `adjustment_range_risk` | 被调剂到非偏好专业的风险 |
| `crowding_adjusted_prob` | 考虑热门拥挤后的录取概率 |
| `recommendation_role` | rush_opportunity / target_core / safe_anchor / avoid |

一句话记忆：

> 每个专业组要同时打三类标签：稳定性、机会性、调剂风险。

### 8.5 代码层需要修正的地方

当前实现里已经读取了 `quota`，但需要避免两个问题：

| 当前问题 | 应该修改 |
| --- | --- |
| `quota` 参与概率计算，但没有作为 `MajorGroupRow` 字段输出 | 在 `MajorGroupRow` 中保存 quota 和 quota_bucket |
| 小招生规模判断使用 `major_count <= 3` | 改成基于招生计划人数 `quota` 判断 |
| 小计划只被当成惩罚 | 同时计算 volatility risk 和 opportunity score |
| 调剂风险主要看专业数量 | 应加入专业组内部专业差异和黑名单占比 |
| 报告只解释录取概率 | 应解释“为什么这是稳定保底 / 波动捡漏 / 拥挤风险” |

推荐算法上的改法：

> 小计划专业组不应该简单降权，而应该分流到“机会候选池”；大计划专业组不应该简单加权，而应该作为“稳定候选池”。最终 portfolio 同时需要 stable anchors 和 variance opportunities。

一句话记忆：

> 小计划进机会池，大计划进稳定池，最后做组合而不是单点排序。

### 8.6 面试里怎么讲这个优化

推荐回答：

> 后来我意识到录取风险不能只看学校和往年最低位次，还要看专业组和招生计划。名额多的专业组更像高流动性资产，历史线通常更稳定，适合做稳妥和保底；名额少的专业组波动更大，可能产生捡漏，也可能产生滑档。所以我会把 quota 拆成两个信号：稳定性和机会性。系统不直接惩罚小计划，而是把它标成 high-variance opportunity，再结合历史波动、热度拥挤和用户偏好决定是否纳入冲刺候选。

一句话记忆：

> 招生少不是坏，招生多也不是绝对好；关键是它在组合里扮演机会还是安全垫。

## 9. 可以放进面试的一段话

> 我后来重新理解这个项目：高考志愿不是普通推荐问题，而是一个机制约束下的高风险匹配市场。文献里用合理妒忌衡量匹配稳定性，说明学生不只关心能不能录取，也关心自己是不是因为机制、信息或策略被错配。平行志愿改革确实降低了合理妒忌和复读，但中分段、低信息家庭和热门专业拥挤仍然有风险。所以我的系统不只做录取概率预测，还希望建一个 decision harness：先解释所在省份的匹配机制，再基于历史数据生成专业组候选，同时识别热门专业的拥挤风险、用户对院校专业的误解和情绪驱动的后悔风险，最后用 critic 检查保底、调剂和证据充分性。

一句话记忆：

> 大部分系统问“你能不能上”，我的系统还问“你为什么想上、这个想法是否被机制误解和群体情绪放大”。
